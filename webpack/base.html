# webpack的5个基本概念

# 提供一个webpack.config.js作为配置文件

其中可以通过 --config 来指定配置文件

// webpack.config.js
module.exports = {
    entry: './src',
    output: {
        path: './dist',
        filename: 'bundle.js'
    },
    module: {
        rules: [

        ]
    },
    plugins: [

    ]
}

  • entry 文件入口 默认值./src

  • output 告诉webpack在哪里输出bundle, 以及如何命名这些文件

    • path 默认值： ./dist
    • filename: bundle.js

  • loader 处理非js文件，并转化为webpack能够处理的文件

    • test 属性, 匹配需要处理的文件
    • use 属性, 表示进行转换时需要用到哪些插件， 从后向前执行

  • plugin loader 被用于转换某些类型的模块，而插件则可以用于执行范围更广的任务。插件的范围包括，从打包优化和压缩，一直到重新定义环境中的变量。插件接口功能极其强大，可以用来处理各种各样的任务。

  • mode development / production
